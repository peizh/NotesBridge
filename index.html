Proof, not pitch

NotesBridge does two concrete jobs.

Editing layer

Make the macOS editing experience less bare.

  • Slash commands with inline exact-match execution
  • Markdown-style triggers for lists and structure
  • Floating formatting toolbar after text selection

Sync output

Export a workspace that still makes sense outside Apple Notes.

  • Folder structure preserved as local files and folders
  • Native attachments exported alongside the note content
  • Front matter and internal note links retained

Who it is for

People who capture in Apple Notes and organize somewhere more durable.

If Apple Notes is where notes arrive and local Markdown is where notes become long-term material, NotesBridge is the bridge between those two systems.

Workflow

Simple to adopt if you already use Apple Notes.

  1. 01

    Install the direct-download build and grant the requested macOS permissions.

    NotesBridge runs quietly in the menu bar and asks for the permissions required for inline enhancements and full-note sync.

  2. 02

    Choose the Apple Notes data folder on the first full sync.

    That allows NotesBridge to read the Apple Notes database and attachment files directly from group.com.apple.notes.

  3. 03

    Keep writing in Apple Notes while the exported workspace stays local.

    The synced result is easier to inspect, search, version, and hand to AI agents or local-first note tools afterward.

Current constraints

Conservative where Apple Notes stays closed.

  • Apple Notes exposes no public plugin API, so NotesBridge is a companion app rather than an in-app extension.
  • Inline enhancements depend on Accessibility and event synthesis.
  • The current sync direction is Apple Notes -> local folders, with no reverse sync.
  • Slash commands currently focus on exact command plus space and mouse-driven suggestion selection.

Q&A

The practical questions people ask first.

Is NotesBridge a plugin inside Apple Notes?

No. Apple Notes does not expose a public plugin or extension API, so NotesBridge runs as a separate macOS companion app.

Does it sync back into Apple Notes?

Not today. The current design is intentionally one-way: Apple Notes -> local Markdown folders.

Why not just export from Apple Notes manually?

The native export path is weak for ongoing workflows. NotesBridge is built around repeated sync, folder preservation, attachment export, and keeping the output useful for local-first tooling.

Does it work with shared notes?

Yes. Shared Apple Notes are part of the intended workflow. The goal is to turn that shared input into a Markdown workspace that stays local and easier to process afterward.

What survives the sync?

The current export keeps folder structure, native attachments, front matter metadata, and internal links. That is the core product shape.
