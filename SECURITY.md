# Security Policy

## Supported Versions

NotesBridge is under active development. Security fixes are expected to land on the current default branch and in the latest tagged release, if releases are available.

## Reporting a Vulnerability

Do not report security vulnerabilities in public GitHub issues.

Use GitHub's private vulnerability reporting / Security Advisories flow when it is enabled for the repository. If that private reporting path is unavailable, contact the maintainer through a private GitHub contact channel instead of opening a public issue.

When reporting a vulnerability, include:

- affected version or commit
- reproduction steps
- impact assessment
- any logs, screenshots, or proof-of-concept details needed to reproduce safely

You will receive an acknowledgement as soon as practical, followed by triage and a fix plan if the report is confirmed.
